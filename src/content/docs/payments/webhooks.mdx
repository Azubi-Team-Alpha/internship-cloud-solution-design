AlphaPay sends webhook events to your registered endpoint URL whenever a notable transaction event occurs. This allows your application to react in real time without polling.

## Supported Events

| Event | Description |
|---|---|
| `transfer.success` | A transfer was successfully delivered to the recipient |
| `transfer.failed` | A transfer failed — check `failure_reason` in payload |
| `payment.received` | A customer payment was successfully collected |
| `payment.failed` | A payment collection attempt failed |
| `transaction.flagged` | A transaction was flagged for AML review |
| `wallet.funded` | A wallet top-up was successfully completed |
| `card.transaction` | A virtual card transaction was authorized |

## Webhook Payload Format

```json
{
  "event": "transfer.success",
  "transfer_id": "txn_GH_20240715_a8f3c",
  "amount": 500,
  "currency": "GHS",
  "recipient_phone": "+233241234567",
  "network": "mtn",
  "timestamp": "2024-07-15T10:24:02Z",
  "signature": "sha256=abc123..."
}
```

## Setting Up Webhooks

<Steps>
1. Go to **Dashboard → Developer → Webhooks** and click **Add Endpoint**

2. Enter your publicly accessible HTTPS endpoint URL

3. Select the events you want to receive and save

4. Copy the **Webhook Secret** shown after saving — you'll use this to verify signatures

</Steps>

## Verifying Webhook Signatures

<Aside type="caution">
  Always verify the webhook signature before processing any event. Unverified webhooks are a security risk.
</Aside>

```javascript
const crypto = require('crypto');

function verifyWebhook(payload, signature, secret) {
  const hmac = crypto
    .createHmac('sha256', secret)
    .update(JSON.stringify(payload))
    .digest('hex');
  return `sha256=${hmac}` === signature;
}
```

## Retry Behaviour

If your endpoint returns a non-2xx response or times out (>30s), AlphaPay will retry the webhook up to **5 times** with exponential backoff: 5s, 30s, 5min, 30min, 2h.
