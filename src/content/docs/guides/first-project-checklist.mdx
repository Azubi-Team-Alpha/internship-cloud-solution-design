Use this checklist to ensure your AlphaPay integration is complete, secure, and production-ready before going live.

## Pre-Integration Setup

<CardGrid>
  <Card title="Account & KYC">
    - Create your AlphaPay Business account at alphapay.africa
    - Complete business KYC (RC number, directors ID, bank statement)
    - Activate your business account — approval in under 24 hours
  </Card>
  <Card title="API Keys & Environments">
    - Retrieve your **test-mode** API key from the Developer dashboard
    - Set up a `.env` file with `ALPHAPAY_SECRET_KEY` and `ALPHAPAY_PUBLIC_KEY`
    - Confirm test-mode transfers work in the sandbox environment
  </Card>
  <Card title="Webhook Configuration">
    - Register a publicly accessible webhook endpoint URL in your dashboard
    - Implement signature verification using your webhook secret
    - Test webhook delivery using the dashboard's "Send Test Event" feature
  </Card>
</CardGrid>

## Integration Steps

### Backend Integration
<Steps>
1. Install the AlphaPay SDK for your platform (Node.js, Python, or PHP) or use raw REST calls

2. Implement the **Initiate Transfer** endpoint for outbound payments

3. Implement the **Payment Collection** endpoint or embed the hosted checkout for inbound payments

4. Set up your webhook handler to process `transfer.success`, `transfer.failed`, and `payment.received` events

</Steps>

### Going Live

<Aside type="caution">
  Before switching to live-mode API keys, ensure all webhook signature validations are in place and you have tested failure scenarios (insufficient funds, invalid account) in sandbox.
</Aside>

<Steps>
1. Replace test-mode API keys with live-mode keys in your production environment variables

2. Set transaction limits appropriate to your expected volumes

3. Verify KYC and compliance documentation is fully approved for your business account

4. Monitor your first live transactions via the AlphaPay merchant dashboard

</Steps>
