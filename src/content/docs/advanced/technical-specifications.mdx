## Bulk Disbursement Optimization

For high-volume payroll or mass payout use cases, use the `/v1/bulk-transfers` endpoint to submit up to 1,000 recipients in a single API call.

<Steps>
1. **Prepare the recipient list**
   Build a JSON array of recipient objects with `phone`, `amount`, `currency`, and `network` fields. Validate all phone numbers before submission.

2. **Submit the bulk transfer**
   POST the recipient array to `/v1/bulk-transfers`. The API returns a `batch_id` immediately — processing happens asynchronously.

3. **Monitor batch progress**
   Poll `GET /v1/bulk-transfers/{batch_id}` or listen for `transfer.success` and `transfer.failed` webhook events per recipient.

4. **Handle partial failures**
   The API returns a per-recipient status. Retry only the failed recipients using the `retry_batch` endpoint — successful recipients are not re-processed.
</Steps>

## FX & Multi-Currency Handling

AlphaPay supports wallets in GHS, NGN, KES, UGX, RWF, and USD. When initiating a cross-border transfer:

- Exchange rates are locked at the time of transfer initiation
- The sender sees the exact destination amount before confirming
- FX rates are sourced in real-time and updated every 60 seconds
- Use `GET /v1/fx-rates?from=GHS&to=NGN` to fetch the current rate before presenting it to users

## AML & Compliance Integration

AlphaPay performs automated AML screening on every transaction. Your integration should:

<Steps>
1. Collect and submit accurate sender and recipient information — incomplete data causes transaction delays

2. Implement the webhook handler for `transaction.flagged` events — flagged transactions require manual review before release

3. For business accounts processing > $50,000/month equivalent, contact your account manager for enhanced due diligence (EDD) onboarding

</Steps>
